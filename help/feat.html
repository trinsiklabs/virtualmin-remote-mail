Use remote mail server?
When this feature is enabled, Virtualmin will configure a remote mail server to handle all email for the domain. This includes:

  • DNS records (MX, SPF, DKIM) pointing to the remote mail server
  • Postfix transport maps on the remote server for mail routing
  • Dovecot mail accounts on the remote server
  • OpenDKIM signing configuration for outgoing email
  • SSL certificate synchronization for encrypted connections
This feature cannot be used together with the local Mail feature, as they would conflict. The DNS feature must also be enabled.
